6 /* FILE: nak.h Specs for CFDP Nak module.
7  * ORIGINAL PROGRAMMER: Tim Ray 301-286-0581
8  * BIGGER PICTURE: In the formal CFDP protocol, the Nak is a Protocol Data
9  * Unit indicating requests for retransmissions. Within this software
10  * library, the Nak structure has the same contents as a Nak PDU, but it
11  * is used in a broader way. The Sender uses the Nak structure to indicate
12  * what data needs to be sent (throughout the life of the transaction, not
13  * just when feedback is received from the Receiver). (The Receiver uses
14  * the Nak structure in the usual way).
15  * SUMMARY: This module maintains NAK-Pdu data (represented as a C structure);
16  * i.e. it keeps track of:
17  * 1) whether or not Metadata needs to be retransmitted
18  * 2) any gaps in the file-data (which File-data needs to be retrans).
19  * This module is designed so that separate transactions can maintain
20  * separate Nak-data (the Nak-data is passed in/out with each routine).
21  * NOTE: Only 'deferred' (comprehensive) Nak-mode is supported!
22  * USAGE:
23  * Call 'nak__init' once for each transaction before calling any of the
24  * other routines.
25  * When copying a Nak structure, use the copy_nak routine below (unless
26  * you enjoy looking through core dumps).
27  * CHANGES:
28  * 2007_07_31 Tim Ray
29  * - Official release. (see pre_release_changes.log for earlier changes)
30  */
