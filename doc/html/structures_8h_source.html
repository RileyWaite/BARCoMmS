6 /* FILE: structures.h -- private data structures; public data structures
7  * are defined in 'cfdp_data_structures.h'.
8  * ORIGINAL PROGRAMMER: Tim Ray 301-286-0581
9  * SUMMARY: (PDU = Protocol Data Unit; i.e. a CFDP message)
10  * This file defines a single data structure that can hold any CFDP PDU,
11  * plus data structures to represent various CFDP concepts.
12  * Each PDU contains a header plus data, where data is either
13  * metadata (MD), file-data (FD), EOF, ack, nak, etc. I have defined
14  * one structure to hold the pdu-header, structures to hold what follows
15  * the pdu-header (i.e. one structure each for metadata, file-data,
16  * EOF, etc), and one "bottom-line" structure that can contain
17  * any CFDP PDU. (Plus there are "building block" data-types that go
18  * inside PDUs; e.g. the CFDP Condition Code)
19  * PURPOSE: These definitions exist in the hope that their use allows
20  * the CFDP source code to be simpler (and easier to read).
21  * The intent is that each incoming PDU is immediately converted to a
22  * structure (which allows easier access to fields within the PDU).
23  * Similarly, each outgoing PDU is generated using the structure, and is
24  * converted to bits just prior to release.
25  * CHANGES:
26  * 2007_07_31 Tim Ray (TR)
27  * - Official release. (see pre_release_changes.log for earlier changes)
28  * 2007_11_19 TR
29  * - Bug fix. 'max_pdu_header_length' calculation was fixed.
30  * (Previously, assumption was that entity-id length = trans-num length)
31  */

139 /* If there are lots of gaps, they may not all fit in one NAK PDU.
140  * This next macro defines the max gaps that will fit in one Nak PDU.
141  * To understand the calculation below:
142  * 1) The part of a CFDP PDU that follows the PDU-header is called the
143  * 'PDU data field'.
144  * 2) The PDU data-field for a Nak PDU has 9 bytes, plus 8 bytes per gap.
145  * (Byte #1 specifies that this is a Nak PDU)
146  * (Bytes #2-9 specify the start-of-scope and end-of-scope;
147  * i.e. to what portion of the file does this Nak apply?)
148  * (Each additional 8 bytes specify start-offset and end-offset of a gap)
149  */
150 #define MAX_GAPS_PER_NAK_PDU ((MAX_PDU_DATA_FIELD_LENGTH - 9) / 8)
