6 /* FILE: aaa.h Specs for CFDP state table 'action' routines.
7  * ORIGINAL PROGRAMMER: Tim Ray 301-286-0581
8  * BACKGROUND:
9  * My philosophy with this module is different than usual.
10  * My usual approach is to make lower-level modules as close to general-
11  * purpose utilities as practical. Instead, this module is tightly coupled
12  * to the needs of the state table modules (so that the state table logic
13  * is easier to grasp).
14  * The purpose of this module is to provide the details for carrying out
15  * many conceptually straightforward actions that are called out within
16  * the CFDP state tables (see s1.c, s2.c, r1.c, and r2.c).
17  * All of the routines in this module work on one state machine at a time.
18  * In most cases, a pointer to the state machine is supplied by the caller.
19  * In many cases, the routine interface could have been more specific
20  * (instead of passing the entire state machine), but that would have made
21  * the state table logic less readable.
22  * CHANGES:
23  * 2007_07_31 Tim Ray (TR)
24  * - Official release. (see pre_release_changes.log for earlier changes)
25  * 2007_11_05 TR
26  * - Removed the routine that retained incomplete files (this routine
27  * is now private).
28  */
